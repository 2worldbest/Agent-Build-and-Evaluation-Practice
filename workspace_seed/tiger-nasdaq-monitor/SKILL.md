# TIGER 미국나스닥 100 모니터링

이 스킬은 사용자가 "TIGER 미국나스닥 100 가격을 모니터링해줘" 또는 "주가가 5% 이상 변동하면 메일로 알려줘"처럼 요청했을 때 동작한다.

## 목표
- 매일 영업일 오전 10시, 오후 3시에 종목 가격을 확인한다.
- 전일 종가와 20일선 가격을 비교한다.
- 가격이 5% 이상 차이날 경우 사용자 메일로 알림을 전송한다.
- 알림 전송 전에는 확인/중단 여부를 사용자에게 알려준다.

## 실행 방식
1. 필요한 데이터를 가져오기 위해 `execute` 도구로 스크립트를 실행한다.
2. 결과를 해석해 기준치(전일 종가, 20일선)와 비교한다.
3. 조건을 만족하면 메일 발송을 시도한다.
4. 메일 발송 결과를 사용자에게 요약해 보고한다.

## 권장 구현 포인트
- 종목 코드: `133690`
- 시장/심볼: `TIGER 미국나스닥 100`
- 실시간 데이터 소스: Yahoo Finance chart API
- 한국 종목 코드는 자동으로 `133690.KS` 형태로 변환한다.
- 비교 기준:
  - 전일 종가
  - 20일선 가격
- 임계값: `5%` 이상 차이
- 알림 채널: 텔레그램 봇
- 환경변수: `YAHOO_API_BASE_URL`, `YAHOO_API_TIMEOUT`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, `EMAIL_RETRIES`, `EMAIL_RETRY_DELAY`
- 텔레그램 메시지 전송 실패 시 기본 3회 재시도, 각 재시도 간 10초 간격으로 처리한다.

## 예시 요청
- 사용자: "TIGER 미국나스닥 100 모니터링 시작해줘"
- 사용자: "오전 10시와 오후 3시에 확인해서 5% 이상 차이나면 메일로 알려줘"
- 사용자: "현재 가격이 20일선보다 5% 이상 차이나면 알려줘"

## 스케줄러 실행
- 스케줄러 스크립트: `python3 workspace/skills/tiger-nasdaq-monitor/schedule_tiger_monitor.py`
- 동작: 매일 영업일 오전 10:00, 오후 15:00에 실행한다.
- 휴일/주말: 실행하지 않는다.
- 공휴일 목록은 스크립트 내부의 `KOREA_HOLIDAYS` 상수로 관리한다.

## 주의사항
- 실제 주가 데이터는 외부 API 또는 웹 스크래핑 기반으로 수집한다.
- 네트워크 오류·API 한도·인증 실패가 발생하면 에이전트는 사용자에게 경고하고 재시도 여부를 제안한다.
- 이메일 발송은 운영 환경의 SMTP/메일 서비스 설정이 필요하다.
