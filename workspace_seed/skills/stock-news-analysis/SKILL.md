# 주식 뉴스 분석 스킬

특정 종목/기업의 최신 뉴스와 가격 지표를 근거 기반으로 수집·요약하고, **주가 영향 가능성**을 방향(긍정/부정/중립)과 영향도 점수로 정리해 투자 **참고용** 브리핑을 만든다. 이 스킬은 투자 자문이 아니며, 매수·매도·보유를 추천하지 않는다.

## 언제 쓰나

- "삼성전자 최근 뉴스 분석해줘", "테슬라 최신 뉴스 알려줘" 같은 종목/기업 뉴스 요청
- "SK하이닉스 개장 전 뉴스 브리핑", "AI 반도체 관련 주요 뉴스 요약" 같은 산업/종목 동향 정리
- 뉴스가 주가에 어떤 영향을 줄지 근거와 함께 판단이 필요할 때

## 평가 기준(반드시 만족)

답변은 아래 5가지 기준을 모두 만족하도록 작성한다.

- **정확성(Correctness)**: 수치·사실·출처가 정확하고, 추정과 사실을 구분한다.
- **관련성(Relevance)**: 종목/이슈와 직접 연결되는 정보만 다룬다.
- **최신성(Freshness)**: 최근 뉴스와 최신 시각 기준으로 정리한다.
- **근거성(Groundedness)**: 각 핵심 문장은 원문/출처 URL과 연결된다.
- **분석력(Insight)**: 단순 요약이 아니라 왜 주가에 영향을 줄 수 있는지 의미 있는 해석을 제공한다.

## 필수 처리 규칙

1. **가격 지표를 반드시 포함한다**
   - **현재가**, **전일 종가**, **전일 대비(%)**, **20일 이동평균선**, **20일선 대비(%)**, **데이터 기준 시각**을 반드시 포함한다.
   - 수치를 조회하지 못하면 추측하지 말고 **조회 실패**로 명시한다. 원인과 다음 확인 방법도 함께 적는다.
   - 가능하면 가격 데이터의 기준 시각과 뉴스의 기준 시각을 구분해 표시한다.

2. **뉴스 수집 (웹 검색 필수)**
   - 웹 검색 커넥터(Tavily 등)를 **최소 2~3회**, 키워드를 바꿔가며 호출한다.
   - 예) "<기업명> 실적", "<기업명> 공급망/수주", "<기업명> 규제/정책", "<티커> news".
   - 금융 토픽은 `topic="finance"`가 유효하다(단 `search_depth`는 `advanced` 사용).
   - 각 기사마다 제목 / 발행일(추정 시 표기) / 출처명 / URL / 핵심 사실을 기록한다.

3. **중복 제거·선별**
   - 같은 사건을 다룬 기사는 하나로 묶는다(원문에 가까운 출처 우선).
   - 대상 종목과 관련성이 낮은 일반 뉴스는 제외한다.
   - 상위 5~10건을 영향도 순으로 선정한다.

4. **영향 분석 (항목별)**
   - **관련성**: 대상 종목과 어떻게 연결되는가(직접/간접 구분).
   - **영향 방향**: 긍정 / 부정 / 중립. 단정 표현("반드시 오른다")은 쓰지 않는다.
   - **영향도**: 1~5점 + 한 줄 근거(왜 그 점수인지).
   - 추정과 사실을 분리하고, 미확인 내용은 `[확인 필요]`로 표기한다.

5. **브리핑 작성**
   - `templates/briefing.md` 형식을 따른다.
   - 핵심 요약을 맨 앞에 둔다.
   - 길거나 재사용될 것 같으면 `write_file`로 `workspace/`에 저장하고 경로를 알린다.

6. **자기 점검 및 보완**
   - 답변 마지막에 반드시 **"평가 기준 체크"** 섹션을 추가한다.
   - 각 기준(정확성·관련성·최신성·근거성·분석력)에 대해 한 줄씩 점검하고, 부족하면 즉시 보완한다.
   - 만약 데이터 조회가 불안정하거나 실패하면 답변을 끝내기 전에 추가 조회를 시도하고, 그래도 안 되면 **조회 실패**를 명시한 뒤 답변을 마무리한다.

## 반드시 지키는 규칙 (요약)

- **출처 없는 항목은 브리핑에 넣지 않는다.** 모든 항목에 URL 필수.
- **원문에 없는 숫자·사실·발언을 지어내지 않는다.** 불확실하면 `[확인 필요]` 또는 **조회 실패**로 표기한다.
- **매수/매도/보유를 추천하지 않는다.** 주가 방향을 단정하지 않는다.
- **영향 방향·영향도에는 반드시 근거를 붙인다.**
- **루머는 넣지 않거나 `[확인 필요]`로만.** 미공개 내부정보를 추정·생성하지 않는다.
- **한국어로 간결하게.** 고유명사(기업명/모델명)는 원문 표기 유지.

자세한 규칙과 템플릿은 `references/rules.md`, `templates/briefing.md` 참조.
